# 光核安利漂流瓶 — 技术框架说明

## 一、项目简介

一个游戏安利主题的 H5 社交活动网站。用户可以投递"漂流瓶"（游戏安利卡片）、随机打捞别人的瓶子、收藏到个人安利墙、分享拉新获取奖励。

---

## 二、技术选型

| 层级 | 技术 |
|------|------|
| 后端 | Python 3.12 + Flask（轻量 Web 框架） |
| 数据库 | SQLite + SQLAlchemy ORM（对象关系映射） |
| 前端 | 原生 HTML/CSS/JS + Jinja2 模板引擎 |
| 图片处理 | Pillow（自动生成缩略图） |
| 用户认证 | Flask-Login（会话管理） |
| 总代码量 | 约 2900 行 |

## 四、数据库设计（8 张表）

### 表关系图

```
users（用户表）
  ├── 基础：id, email(唯一), nickname(唯一), password_hash
  ├── 拉新：referral_code(分享链接标识), referrer_id(邀请人ID)
  ├── 权限：is_admin
  └── 关联：一对多 → bottles, salvage_records, daily_tasks

drift_bottles（漂流瓶表）
  ├── 内容：title, game_name, field_a, field_b, recommendation
  ├── 分类：category（7大分类ID）
  ├── 图片：image_path, thumbnail_path
  ├── 审核：is_approved, is_preset, review_note
  └── 外键：user_id → users

salvage_records（打捞记录表）
  ├── 外键：user_id → users, bottle_id → bottles
  ├── 安利墙：is_saved_to_wall
  ├── 唯一约束：同一用户不能重复打捞同一个瓶子
  └── 时间：salvaged_at

bottle_likes（点赞表）
  ├── 外键：user_id → users, bottle_id → bottles
  └── 唯一约束：同一用户只能点赞一次

daily_tasks（每日任务表）
  ├── 外键：user_id → users
  ├── 日期：task_date
  ├── 任务状态：throw_claimed, share_claimed
  ├── 拉新计数：referral_count
  └── 唯一约束：每个用户每天一条记录

share_records（分享记录表）
  ├── 外键：sharer_id → users, new_user_id → users
  ├── 分享类型：shared_type（activity/bottle/wall）
  └── 时间：created_at

promo_codes（推广码表）
  ├── code(唯一), channel_name, max_uses, use_count
  └── 使用记录：used_by, used_at

admin_logs（管理员操作日志）
  └── admin_id, action, target_type, note
```

## 五、核心业务逻辑

### 5.1 漂流瓶生命周期

```
投递 → 敏感词检测 → 审核（自动/人工）→ 进入漂流池
                                          ↓
                                    用户随机打捞
                                          ↓
                              查看详情 / 点赞 / 收藏到安利墙
                                          ↓
                                    分享给好友
```

### 5.2 打捞算法

1. 优先抽取非预设瓶（真实用户投递的）
2. 排除自己投的瓶子
3. 排除已经捞过的瓶子
4. 随机排序取出一个
5. 如果非预设瓶没了，回退到预设瓶

### 5.3 任务与奖励

```
每日基础打捞次数：1 次

完成任务可增加次数：
  投递一个瓶子   → +1 次
  分享活动/瓶子   → +1 次（每日首次自动领取）
  拉新成功(每人) → +2 次（每日最多2人）

每日总上限：可配置（当前1000次）
每日 0:00 重置
```

### 5.4 分享拉新机制

**分享方式**
- 手机端：调用系统原生分享面板（微信/朋友圈/QQ）
- 桌面端：复制链接到剪贴板

**链接格式**（每人专属）
```
https://域名/?via=用户名          → 活动首页
https://域名/s/bottle/3?via=用户名 → 某个瓶子
https://域名/s/wall/5?via=用户名   → 某人安利墙
```

**拉新判定**
- 每个用户有唯一 `referral_code`（默认=用户名）
- 新用户通过 `?via=xxx` 链接注册 → 自动计入 xxx 的拉新奖励
- 注册时 `referrer_id` 记录邀请人，`DailyTask.referral_count` +1

## 六、7 大内容分类

每个漂流瓶属于一个分类，不同分类有不同的填写字段：

| 分类 | 字段A | 字段B |
|------|-------|-------|
| 📸 炸裂截图 | 这张图发生的前提 | 最绝的地方在于 |
| 🌅 绝美场景 | 打卡地/时机 | 这画面神在哪里 |
| 💖 人物塑造 | TA是谁 | 为什么印象深刻 |
| 🎨 美术鉴赏 | 皮肤/外观名 | 设计最戳我的点 |
| 🎬 剧情放映 | 这是什么情节 | 当时的心理活动 |
| ⚔️ 硬核博弈 | 当时的境况 | 决定性瞬间 |
| 🌟 屏幕外回响 | 经历时的现实背景 | 游戏/人给了我什么 |

## 七、安全与防刷

| 措施 | 说明 |
|------|------|
| CSRF 防护 | 所有表单提交需携带动态 token，防止跨站请求伪造 |
| 密码加密 | Werkzeug 哈希存储，不明文保存 |
| 敏感词过滤 | 投递时自动检测敏感词词库，命中则标记待审 |
| 内容审核 | 管理员后台可人工通过/驳回/删除 |
| 投递冷却 | 10 秒内不能重复投递 |
| 打捞冷却 | 2 秒内不能重复打捞 |
| 重复打捞 | 数据库唯一约束，同一用户不能捞同一个瓶子两次 |
| 图片限制 | 仅允许 JPG/PNG/WebP，最大 20MB |
