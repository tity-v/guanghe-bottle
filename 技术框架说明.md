# 光核安利漂流瓶 v5 — 技术框架

## 技术栈

| 层 | 技术 |
|---|---|
| 后端 | Python 3.12 + Flask |
| 数据库 | SQLite + SQLAlchemy ORM |
| 前端 | 原生 HTML/CSS/JS + Jinja2 |
| 图片 | Pillow（缩略图） |
| 认证 | Flask-Login（QQ号登录，无密码） |

## 项目结构

```
├── app.py              # 控制器，全部路由+业务逻辑
├── models.py           # 7 张表模型
├── config.py           # 活动规则 + 7 大分类
├── sensitive_words.txt # 敏感词库
├── wsgi.py             # 生产入口（Gunicorn）
├── railway.toml        # Railway 部署配置
├── requirements.txt
├── static/
│   ├── css/style.css
│   ├── js/main.js
│   └── images/         # Logo/背景/按钮等
├── templates/          # 14 个模板
│   ├── base.html       # 父模板
│   ├── index.html      # 首页
│   ├── login.html      # 登录（QQ号）
│   ├── throw.html      # 投递
│   ├── salvage.html    # 打捞
│   ├── wall.html       # 安利墙 + 分享图
│   ├── wall_public.html
│   ├── collection.html # 打捞记录
│   ├── bottle_detail.html
│   ├── shared_bottle.html
│   ├── tasks.html      # 任务
│   └── admin/          # 管理后台（4页）
└── uploads/bottles/    # 上传图片
```

## 7 张数据表

```
users              # id, qq, nickname, is_admin, referral_code, referrer_id
drift_bottles      # 内容+图片+分类+审核状态
salvage_records    # 打捞记录（唯一约束：一人一瓶一次）
bottle_likes       # 点赞（唯一约束）
daily_tasks        # 每日任务（投递/分享/拉新）
share_records      # 分享追踪
admin_logs         # 管理员操作日志
```

## 核心流程

```
投递 → 敏感词检测 → 干净? → 直接上架
                    ↓ 有风险
                  进审核队列 → 管理员通过/驳回

打捞 → 随机抽一个 → 排除自己投的+已捞过的 → 展示信纸卡片
                                                      ↓
                                              点赞 / 收藏到安利墙 / 分享
```

## 认证（v5）

- **登录 = 自动注册**：输入 QQ 号即可
- 新用户自动创建账号，昵称默认用 QQ 号
- 支持 `?via=referral_code` 邀请链接追踪

## 管理员（QQ 10001/10002/10003）

- 每日投递 **99 次**（普通用户 5 次）
- 审核后台：有风险 / 未审核 / 已审核 三栏
- 可对已上架瓶子直接删除

## 任务奖励

| 行为 | 打捞次数 |
|---|---|
| 每日基础 | 1 |
| 投递瓶子 | +1 |
| 分享活动 | +1 |
| 拉新（每人） | +1（日限 2 人） |
| 每日上限 | 5 |

## 安全

- CSRF token 校验所有 POST 请求
- 敏感词库 + 正则（手机号/链接/微信号）双重检测
- 管理员审核 + 驳回/删除
- 投递冷却 10s / 打捞冷却 2s

## 启动

```bash
pip install -r requirements.txt
python app.py          # → http://localhost:5000
```

生产：`gunicorn wsgi:application`（Railway 自动部署）
